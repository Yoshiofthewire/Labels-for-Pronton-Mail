# Auth Setup How-To

This guide explains how to collect the credentials and files needed for this project to access Proton Mail and Lumo.

## What You Need

- Lumo auth session file for local Lumo API V2:
  - /lumo_lab/config/lumo-auth.json
- Proton token runtime file (generated by this app):
  - /lumo_lab/config/proton-auth.json

## 1) Lumo: Create the Auth File

This project runs local Lumo API V2 in the same container. It expects a valid auth session file.

Required path inside the container:
- /lumo_lab/config/lumo-auth.json

How to get it:
1. Follow the login/auth steps from the Lumo API V2 project to generate its auth.json.
2. Save the generated file on your host.
3. Start this app normally.
4. Open the Configuration page in the web UI.
5. In the `Lumo Authentication` section, upload the generated auth.json file.

In this repo, the backend stores the uploaded file at `/lumo_lab/config/lumo-auth.json` and restarts only the local Lumo process so the new auth file is copied into the Lumo runtime immediately.

## 2) Proton: Convert auth.json to Runtime Tokens

Use the same generated auth.json to populate Proton API runtime tokens for the Go backend.

1. In Configuration page, go to `Proton Authentication`.
2. Upload the generated auth.json file.
3. The backend extracts token data from Proton `AUTH-*` and `REFRESH-*` cookies and writes:
   - /lumo_lab/config/proton-auth.json
4. If conversion succeeds, restart the app container (or use repair) so daemon picks up the new Proton token file.

Optional:
- Set PROTON_API_HOST only if you need a non-default Proton API host.

## 3) Fill Your .env File

At minimum, set:
- LUMO_API_KEY (if your Lumo route requires it)
- LUMO_BASE_URL (default is local: http://127.0.0.1:3333)
- LUMO_LOCAL_ENABLED=true (for local Lumo process)
- PROTON_AUTH_FILE=/lumo_lab/config/proton-auth.json

Then start:

- docker compose up --build -d

## 4) Verify in UI

1. Open the web UI at http://localhost:5866
2. Log in with admin credentials from /lumo_lab/config/admin.env (created on first run).
3. Open Configuration page.
4. Run Lumo Test.
5. If needed, use logs endpoint to troubleshoot:
   - /api/logs?lines=200

You can also use the `Refresh Lumo Auth Status` button to confirm the auth file is present before testing.

## Troubleshooting

- Proton auth fails:
  - Confirm conversion in `Proton Authentication` completed successfully.
  - Confirm /lumo_lab/config/proton-auth.json exists and is readable.
- Lumo local process idle or fails:
  - Confirm lumo-auth.json was uploaded successfully and exists at /lumo_lab/config/lumo-auth.json.
  - Check container logs for lumo process startup messages.
- Labels are not applied:
  - Verify Proton credentials are valid.
  - Ensure labels.allowlist is not empty.
  - Check daemon and API logs.

## Security Tips

- Never commit real credentials or tokens.
- Keep .env private.
- Rotate passwords/tokens if exposed.
